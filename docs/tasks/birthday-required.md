# Задача: Обязательное заполнение поля ДР в ЛК

**Статус:** В работе
**Дата:** 2026-04-26

## Описание

Сделать обязательным заполнение поля "Дата рождения" в ЛК.

## Сценарий

Сайт → ЛК → Настройки профиля → поле "Дата рождения"

## План / Результат

### Логика:

1. **Подсветка пустого поля ДР** — если ДР не заполнено, колонка с датапикером подсвечивается (highlight-бордер)
2. **Поп-ап на 5 сек** — при входе в настройки профиля с пустым ДР выводить поп-ап: _"Заполни дату рождения, чтобы получить бесплатный подарок от команды «Again»"_
3. **Заполнение ДР** — пользователь заполняет дату рождения через датапикер
4. **Сохранение** — пользователь нажимает "Сохранить изменения"
5. **Блокировка поля** — после сохранения поле ДР становится неактивным (disabled), редактирование невозможно
6. **Подарок на ДР** — за 3 дня до ДР пользователь получает промокод (существующая логика `birthday:process`)

## Затронутые файлы

### Фронт (nuxt-shop)

| Файл | Изменение |
|------|-----------|
| `components/Form/Datepicker.vue` | + props `disabled`, `highlight` + стили |
| `components/Modals/BirthdayReminder.vue` | **Новый** — поп-ап с призывом заполнить ДР (5 сек авто-закрытие) |
| `pages/profile/settings.vue` | + логика: подсветка пустого ДР, поп-ап при onMounted, disabled после сохранения |

### Бек (laravel)

| Файл | Изменение |
|------|-----------|
| `app/Http/Controllers/Api/Admin/ClientController.php` | В `update_profile()` — игнорировать birthday если уже заполнено |

## Технические детали

### Фронт

- `FormDatepicker` получает новые props:
  - `disabled: boolean` — блокирует VueDatePicker
  - `highlight: boolean` — добавляет класс `_highlight` с подсветкой бордера
- В `settings.vue`:
  - `isBirthdayFilled` = `!!user?.profile?.birthday` (данные с сервера)
  - `onMounted()`: если ДР пустое → `highlight=true` + показать `BirthdayReminder` модал
  - После успешного `save()`: `disabled=true`, `highlight=false`
- `BirthdayReminder.vue` — аналог `ModalsMessage` но с `setTimeout 5000ms`

### Бек

- В `ClientController::update_profile()`:
  - Перед обновлением проверить: если `$client->profile->birthday` уже заполнено — убрать `birthday` из данных для обновления (молча игнорировать)
  - Остальные поля сохраняются как обычно

## Чек-лист проверки

- [ ] Открыть ЛК с пустым ДР → поле подсвечено, поп-ап 5 сек
- [ ] Заполнить ДР, сохранить → поле заблокировано
- [ ] Открыть ЛК с заполненным ДР → поле заблокировано, поп-ап не показан
- [ ] PUT /clients/update-profile с birthday при заполненном ДР → birthday не меняется
- [ ] За 3 дня до ДР — промокод создаётся (существующая логика)
